This is a key handoff document between business and engineering. Use [BRACKETS] for all placeholders.

Sections:

1. Document Header
   - Product Name: [PRODUCT NAME]
   - Version: 1.0 Draft
   - Product Manager: [NAME]
   - Engineering Lead: [NAME]
   - Date: [DATE]
   - Status: Draft

2. Product Overview
   - Problem Statement: [2-3 sentences about the pain point]
   - Solution: [What this product does]
   - Target Users: [Primary Persona], [Secondary Persona]
   - Success Metrics (KPIs): table with Metric | Target | Measurement Method

3. Goals & Non-Goals
   - In Scope (bulleted list of 6-8 items)
   - Out of Scope (bulleted list of 4-5 items)

4. User Personas
   For each persona (create 2):
   - Name, Role, Age range, Goals, Pain Points, Tech Savviness

5. Key Features — MVP
   For each feature (list 6-8), provide:
   - Feature name
   - Description
   - User value
   - Priority: [P0 / P1 / P2]
   - Effort estimate: [S / M / L / XL]
   - Acceptance criteria (3-5 bullet points)

6. Feature Table Summary
   - Table: Feature | Priority | Effort | Status | Owner

7. User Flows (Narrative)
   - Primary user journey 1: [Onboarding flow]
   - Primary user journey 2: [Core action flow]
   - Edge cases to handle

8. Functional Requirements
   - Numbered list of specific system behaviors (FR-001 through FR-020)
   - Format: FR-001 | The system shall... | Priority | Notes

9. Non-Functional Requirements
   - Performance: [Page load < X seconds, API response < Xms]
   - Availability: [99.9% uptime SLA]
   - Scalability: [X concurrent users at launch]
   - Accessibility: [WCAG 2.1 AA]
   - Browser/Device support table

10. Dependencies & Risks
    - Table: Dependency | Team | Risk Level | Mitigation

11. Open Questions
    - Numbered list of 8-10 unresolved product decisions

12. Approval Sign-offs
    - Table: Role | Name | Signature | Date

Format cleanly in Markdown with tables and numbered lists throughout.
