INTEGRATION PROMPT
Audit existing approval and notification system. Overlay ONLY missing capabilities:

1. Implement approval state machine: NOT_SUBMITTED → SUBMITTED → APPROVED or REJECTED. Withdraw returns to NOT_SUBMITTED. Rejected entries return to editable state for submitter.
2. Implement audit fields as first-class system data: Submitted On/By, Approved On/By, Rejected On/By. Auto-set on status change. Auto-clear on reversal. Undo approval clears Approved On/By and also clears any Rejected On/By. Never let these become stale. Make available in exports from day one.
3. Implement Approvals Tab: manager/admin view listing team members, submission status, total hours, capacity %, billable %, inline actions (Approve, Reject, Send Reminder, Message, Apply Time Off).
4. Implement bulk operations: select multiple team members → Bulk Approve. Select multiple entries → Bulk Reject with reason.
5. Implement detailed review: click team member name → full weekly grid with per-row data. Approve button approves entire week. Reject applies to selected entries with reason text.
6. Implement in-context messaging: Message button opens conversation thread visible to all approvers and submitter for that timesheet period.
7. Implement all 7 notifications with Email + In-app channels:
   a) Reminder Before Due Date
   b) Reminder After Due Date
   c) Timesheet Approved
   d) Timesheet Rejected
   e) Approval Requested
   f) Manual Nudge/Reminder
   g) Time Off Applied
   All except manual nudge must be individually toggleable by users in notification preferences.
8. Implement approver routing: global setting — either Project Owners approve (participants route to respective project owner) or designated Timesheet Approver per team member. If neither configured, default to account Admins.

- Date-based lock (Phase 3)
- Approver hierarchy (Enterprise)
- Submit on behalf (Phase 5)
