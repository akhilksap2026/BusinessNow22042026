STEP 2 — Backend: task routes (hours model)
File: artifacts/api-server/src/routes/tasks.ts

a) POST /tasks (create)
  - Accept optional plannedHours and estimateHours in body.
  - If plannedHours is provided and estimateHours is NOT, default estimateHours = plannedHours.
  - Persist both. Keep persisting effort = plannedHours for backward compat.

b) PATCH /tasks/:id (update)
  - Accept optional plannedHours and estimateHours. Update whichever is provided.

c) GET /tasks and GET /tasks/:id (read)
  - For each task, compute and include:
      actualHours: sum of time_entries.hours WHERE task_id = task.id (default 0 if none)
      etc:         estimateHours − actualHours  (can be negative)
      eac:         actualHours + Math.abs(etc)
  - Return all 5 fields: plannedHours, estimateHours, actualHours, etc, eac.
  - Keep returning the existing effort field.

File: artifacts/api-server/src/routes/timeEntries.ts
  - POST /time-entries: accept optional taskId (number | null). Persist it.
    Validate: if taskId provided, the task must belong to the same projectId.
  - GET /time-entries: include taskId in the response.

─── NEW (Feedback 1): task notes routes ──────────────────
File: artifacts/api-server/src/routes/tasks.ts
(add after the existing task routes, or in a new tasks-notes.ts
 that is mounted on the same Express router)

Route A: GET /tasks/:id/notes
  - Fetch all task_notes WHERE task_id = :id, ORDER BY created_at ASC.
  - For each note, JOIN users to get the author name:
      userName: usersTable.name where usersTable.id = note.userId
  - Return: array of { id, taskId, userId, userName, content, createdAt, updatedAt }
  - No auth restriction beyond logged-in.

Route B: POST /tasks/:id/notes
  - Body: { content: string, userId: number }
  - Validate: content must be non-empty string, userId must be a valid user.
  - INSERT into task_notes. Return the created note with userName resolved.
  - No PM restriction — any logged-in user can add a note.

Route C: DELETE /tasks/:taskId/notes/:noteId  (requirePM OR note owner)
  - Delete a note by id if the requesting userId === note.userId OR role is PM/admin.
  - Return 204 on success.
