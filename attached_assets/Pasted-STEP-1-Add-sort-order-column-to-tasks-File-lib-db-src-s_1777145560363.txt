Run: cd lib/db && npm run push
No migration needed — all existing tasks get sortOrder = 0.

Backend: artifacts/api-server/src/routes/tasks.ts
- GET /tasks: ORDER BY sort_order ASC, id ASC.
- PATCH /tasks/:id: accept sortOrder and parentTaskId in body.
- Add PATCH /tasks/reorder:
    Body: { updates: Array<{ id, sortOrder, parentTaskId }> }
    Apply all updates in a single DB transaction. Return { updated: N }.
    Auth: requirePM.

STEP 3 — Drag-and-drop in the task list
- Wrap the task list in DndContext + SortableContext.
- Each task row is a useSortable item with id = task.id.
- Add a drag handle (⠿ icon) on the left side of each row.
- On drag end: push the change to pendingChanges (NOT immediately to API).
  Re-render list optimistically. Show the "Unsaved Changes" bar.
  Indent right (>24px) → make child; indent left → promote.
- Render a blue horizontal drop-line indicator during drag.

STEP 4 — Unsaved changes bar + Save/Discard/Undo
- Add state: pendingChanges: Map<number, {sortOrder, parentTaskId}>
  and changeHistory: Array of reversible ops.
- When pendingChanges.size > 0, render a sticky amber bar at the bottom:
    "● X unsaved change(s)  [Save]  [Discard]"
- Save: POST to PATCH /tasks/reorder. On success: clear state, invalidate query.
- Discard: clear state, invalidate query (revert to server state).
- Ctrl+Z / Cmd+Z: pop last changeHistory entry, revert in pendingChanges.
