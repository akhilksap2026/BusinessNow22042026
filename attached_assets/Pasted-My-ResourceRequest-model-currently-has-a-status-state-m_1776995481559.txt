My ResourceRequest model currently has a status state machine 
(Draft → Pending → Approved | Blocked) and basic fields 
(project_id, role_id, requested_by, approver_id, comments).

TASK: Expand resource requests with 4 specific types and 
an improved approver view. Do not change the state machine.

--- STEP 1: Add request_type field ---
Add enum field `request_type` to ResourceRequest:
Values: 'add_member' | 'add_hours' | 'assign_placeholder' | 
        'replacement'

--- STEP 2: Conditional form fields per type ---
In the New Request form, after the user selects request_type, 
show/hide fields accordingly:

'add_member':
  - Role (dropdown), Region (dropdown), Required Skills 
    (multi-select from Skills), Start Date, End Date, 
    Hours/Day, Allocation Method, Notes

'add_hours':
  - Team Member (dropdown — must already be on the project), 
    Additional Hours Needed (number), Date Range, Notes
  - Show current allocated hours for selected member on 
    that project as read-only context

'assign_placeholder':
  - Placeholder (dropdown — show only unfilled placeholders 
    on the selected project), Required Skills, Notes
  - Pre-fill dates from the placeholder's existing allocation

'replacement':
  - Current Member/Placeholder (dropdown), 
    Replacement Start Date, Replacement End Date, 
    Reason (text), Notes
  - Show warning if project has auto_allocate = true: 
    "Replacement requests are not available for 
    auto-allocate projects"
  - Disable submission if auto_allocate is true

--- STEP 3: Relevant Matches panel in approver view ---
When an approver opens a Pending request, add a 
"Relevant Matches" panel on the right side:

- Query team members filtered by: role matching request, 
  skills matching request (if specified)
- For each candidate show:
  - Name, Role
  - Forecasted Utilisation %: 
    (existing_hours + proposed_hours) / capacity × 100
  - Over-allocation warning (in red) if forecasted > 100%
  - "Ignore Soft Allocation" toggle at the top of the panel — 
    when toggled, recalculate utilisation excluding soft 
    allocations from existing_hours
- Clicking a candidate pre-fills the "Assign To" field 
  in the approval action

Do not change the approval state machine or notification logic.
