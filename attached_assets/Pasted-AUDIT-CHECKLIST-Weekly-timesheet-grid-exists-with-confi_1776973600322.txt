INTEGRATION PROMPT
Audit the existing timesheet interface. Overlay ONLY missing capabilities:

1. Implement weekly timesheet grid with configurable calendar start day (Monday or Sunday). Columns = working days + Total. Rows = tasks/activities.
2. Implement Row Persistence via join table (user_id + work_item_id). Once added, a row appears in all subsequent weeks automatically. Do NOT implement as per-week records.
3. Implement three entry methods:
   a) Grid cell entry: click cell, type hours, auto-save. Tab moves to next cell.
   b) Task-level clock: task detail panel opens time entry form (Date, Hours, Billable, Category, Notes), syncs immediately to grid. Multiple entries per task per day allowed.
   c) Add Activity modal: select Project or Non-Project Activity, fill Activity Name, Category, Billable, Notes. Creates persistent ad-hoc row.
4. Enforce Non-Project Activity rule at data model level: if project_id is null, is_billable must be FALSE. This must be a database constraint, not UI-only.
5. Implement entry-level billable boolean on every time entry. Pre-fill from task default or account default, but allow user override.
6. Implement Time Entry Categories: admin-defined list, selectable dropdown on every entry method. Optional unless custom field makes mandatory.
7. Implement basic submission: Submit Week for Approval button validates minimum hours (if configured) and transitions week to Submitted state.
