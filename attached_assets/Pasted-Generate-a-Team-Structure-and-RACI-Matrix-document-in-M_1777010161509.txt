Generate a Team Structure and RACI Matrix document in Markdown format for a SaaS product development team.
Save it to /docs/operations/13_team_structure_and_raci.md

Use [BRACKETS] for all placeholders. This is for the delivery and operations teams.

Sections:

1. Document Header
   - Product: [PRODUCT NAME], Date: [DATE], Owner: [PM / Delivery Lead]

2. Team Overview
   - Core team table: Role | Name | Allocation | Reporting To | Start Date
     Include: Product Manager, Tech Lead, Frontend Dev (x2), Backend Dev (x2),
     UX Designer, QA Engineer, DevOps/Infra, Business Analyst, Scrum Master

3. Org Chart (Text)
   - Simple Mermaid diagram showing reporting lines

4. Role Responsibilities
   For each role, provide:
   - Role title
   - Key responsibilities (4-5 bullets)
   - Skills required
   - Success metrics for this role

5. RACI Matrix
   - Full RACI table across key project activities
   - Rows: Activities (20-25 realistic activities across the SDLC)
   - Columns: PM | Tech Lead | Frontend Dev | Backend Dev | UX | QA | DevOps | Business
   - Values: R (Responsible) | A (Accountable) | C (Consulted) | I (Informed)

   Include activities such as:
   - Product requirements definition
   - Architecture design
   - UI/UX design
   - Sprint planning
   - Feature development
   - Code review
   - Unit testing
   - Integration testing
   - UAT
   - Deployment to staging
   - Deployment to production
   - Bug triage
   - Security review
   - Documentation
   - Stakeholder reporting
   - Vendor/tool selection
   - On-call rotation
   - Retrospectives
   - Roadmap planning
   - Budget tracking

6. Communication Protocols
   - Table: Meeting | Frequency | Attendees | Owner | Format
   - Include: Daily standup, Sprint planning, Sprint review, Retrospective,
     Stakeholder update, Architecture review, All hands

7. Escalation Path
   - Flowchart description: Issue → [First contact] → [Second] → [Executive]
   - SLA for escalation response times

Format with clean tables throughout.
