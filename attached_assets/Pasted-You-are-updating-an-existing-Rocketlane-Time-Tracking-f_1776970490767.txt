ADDENDUM A — Customer Portal: External User Time Logging

Section to add: "Client Portal — External User Time Logging" under the Client Portal 
integration section.

Rocketlane's Customer Portal V2 supports time tracking for external (customer-side) 
users. Key facts to document:

- Admins configure per-project which projects expose time tracking in the customer portal.
- Customers access their own timesheet from within the portal interface (same weekly grid 
  concept, scoped to their portal view).
- Customers log time against project tasks they are responsible for within the portal.
- Time entries submitted by customer/portal users flow through the SAME approval workflow 
  as internal team members — they are not auto-approved.
- This is distinct from the read-only Time Tracking Widget (budgeted vs tracked hours 
  display). This feature allows active time logging, not just viewing.
- Data model implication: the "Team Member" field on a TimeEntry must support portal/
  external user types, not just internal users.
- RBAC implication: portal users have a separate role type. Their time entries are 
  visible to project owners and approvers through the standard Approvals tab and 
  Tracked Time tab.

ADDENDUM B — Time Tracking in Project Templates

Section to add: under "Setup & Configuration" → new sub-section "Template-Level 
Time Tracking Configuration".

When time tracking is enabled account-wide, project templates gain time tracking 
configuration options at the task level. Key facts:

- Navigation: Templates → select relevant project template → click any task → 
  locate "Time Entry Settings" section in the task detail panel.
- Per-task configurable settings in templates:
    (a) Mark task as Billable (yes/no) — becomes the billable default when team members 
        log time against this task in any project created from this template.
    (b) Map task to a specific Time Entry Category — pre-assigns the category for time 
        entries on this task.
- These settings carry forward to ALL projects created from this template, 
  eliminating the need to configure each task individually in every project.
- Template-level billable/category defaults can be overridden per time entry by the 
  team member at logging time.
- Resource allocation placeholders defined in templates (roles + hours) are used as 
  the starting allocation baseline for new projects and feed into Import from 
  Allocations.

ADDENDUM C — Two Missing Global Settings

Add these two rows to the Global Settings table in "Setup & Configuration":

1. Setting name: "Billable Default"
   Options: Billable / Non-Billable
   Default: Billable
   Notes: Account-wide default for the billable flag on new tasks. Applies when no 
   task-level or template-level billable setting is configured. Can be overridden 
   per task.

2. Setting name: "Calendar Start Day"
   Options: Monday / Sunday
   Default: Monday
   Notes: Sets which day is rendered as the first column in the weekly timesheet grid. 
   Affects week boundary calculations for submission deadlines and weekly totals.

ADDENDUM D — Operations Insights: Tracked vs Planned Tab

Section to add: under "Reports & Visualizations" → new sub-section 
"Operations Insights — Tracked vs Planned Tab".

Path: Reporting → Operations Insights → Tracked vs Planned tab.
Plan availability: All plans.

Purpose: Compares tracked time entries against planned (budgeted) hours at the task, 
phase, and project level within a selected project template. Surfaces over-burning and 
under-burning patterns to help refine future estimates in templates.

Key facts to document:
- Scoped to a selected template — allows comparison across all projects that 
  were created from that template.
- Granularity levels: Project level → Phase level → Task level (drill-down).
- Shows: Planned Hours (from template task effort estimates), Tracked Hours 
  (from timesheet actuals), Variance (Planned − Tracked), Variance %.
- Over-burning (Tracked > Planned) typically highlighted in red/amber.
- Under-burning (Tracked < Planned) highlighted in green.
- Use case: If "Customer Onboarding" tasks consistently burn 40% more than planned 
  across 20 projects, this tab surfaces it — enabling template refinement.
- This report connects time tracking data directly to delivery operations 
  improvement, making it one of the highest-value reports for a PS team.

ADDENDUM E — People Performance Report: Time-Tracking Columns

Section to add: under "Reports & Visualizations" → new sub-section 
"People Performance Report — Time Tracking Fields".

Path: Reporting → People Performance. Plan availability: All plans.

This foundational report includes time-tracking-specific columns alongside 
task performance metrics. Document the following fields that are sourced from 
time tracking data:

| Field               | Description                                          | Formula / Source         |
|---------------------|------------------------------------------------------|--------------------------|
| Capacity            | Total available hours in the period (excl. time-off  | Calendar × working hours |

Capacity formula (same as all other time-tracking reports):
  Capacity = Total working hours in period − Time-Off hours − Holiday hours

KPI cards (top of People Performance Report — 3 "Top Performer" spotlight cards):
- Top Performers: Task Completion — top 3 team members by % of assigned tasks 
  marked Completed.
- Top Performers: Billable Utilization — top 3 by billable utilization %.

The People Performance report is the only foundational report that shows 
task completion speed, collaboration activity, AND billable utilization 
in a single view — making it the go-to report for performance reviews.
