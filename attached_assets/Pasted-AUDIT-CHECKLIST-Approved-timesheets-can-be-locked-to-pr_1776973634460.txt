INTEGRATION PROMPT
Audit existing governance controls. Overlay ONLY missing capabilities:

1. Implement Lock-on-Approval: toggle in global settings. When ON, team members cannot withdraw, edit, or resubmit after approval. Admins remain exempt.
2. Implement Date-Based Period Lock: admin sets Lock Date. Entries with date ≤ lock_date are frozen for non-admins. Two independent sub-permissions control override:
   a) Can edit details of date-locked entries
   b) Can change status of date-locked entries
3. Implement Status Lock: when date lock is active, non-admins cannot submit, approve, reject, or withdraw entries dated on or before lock date. The "Can change status of date-locked entries" permission is automatically disabled for non-admins when Status Lock is ON.
4. Preserve deactivated user data: implement soft-delete (is_active = FALSE). All historical TimeEntries, Timesheets, approvals, audit records remain visible in reports. Deactivated users disappear from active dropdowns only.
5. Protect invoiced entries: if is_invoiced = TRUE, block movement to another project. Require invoice void/delete first.
