Generate a professional REST API Documentation template in Markdown format for a SaaS application.
Save it to /docs/technical/02_api_documentation.md

The document must include the following sections. Use [BRACKETS] for all product-specific placeholders.

Sections:

1. Document Header
   - Title, Version, Base URL: https://api.[YOURDOMAIN].com/v1, Auth: Bearer Token (JWT)

2. Overview
   - Brief API purpose
   - Versioning strategy (URI versioning)
   - Rate limiting policy: [X requests per minute per API key]

3. Authentication
   - How to obtain a token (POST /auth/login)
   - How to pass token in headers: Authorization: Bearer {token}
   - Token expiry: [X hours]

4. Error Handling
   - Standard error response format (JSON code block)
   - Error codes table: Code | HTTP Status | Meaning
     Include: 400, 401, 403, 404, 409, 422, 429, 500

5. API Endpoints (generate one full example endpoint block for each of these groups)
   Group them under these resource headings:

   ### Auth
   - POST /auth/register
   - POST /auth/login
   - POST /auth/logout
   - POST /auth/refresh-token

   ### Users
   - GET /users/me
   - PUT /users/me
   - DELETE /users/me

   ### [RESOURCE A - e.g., Projects]
   - GET /[resource]
   - POST /[resource]
   - GET /[resource]/{id}
   - PUT /[resource]/{id}
   - DELETE /[resource]/{id}

   ### [RESOURCE B - e.g., Items/Records]
   - (same CRUD pattern)

   For each endpoint provide:
   - HTTP Method + Path
   - Description
   - Request Headers
   - Request Body (JSON example in code block, with field table below: Field | Type | Required | Description)
   - Response (success JSON code block)
   - Error responses possible

6. Webhooks (if applicable)
   - Events list: [event.created], [event.updated], [event.deleted]
   - Webhook payload example

7. Pagination
   - Standard query params: page, limit, sort, order
   - Response envelope format (data, meta.total, meta.page, meta.limit)

8. Changelog
   - Table: Version | Date | Changes

Use clean Markdown, JSON code blocks, and HTTP method badges in bold.
