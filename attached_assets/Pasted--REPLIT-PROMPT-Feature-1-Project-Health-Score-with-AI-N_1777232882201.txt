### SECTION A — Backend: Health Score Computation Endpoint

**File to create:** `artifacts/api-server/src/routes/projectHealth.ts`

Create a new GET endpoint: `GET /api/projects/:id/health`

This endpoint is protected by the existing RBAC middleware (same permission as `GET /api/projects/:id` — any user who can view the project can call this).

The endpoint computes a health score from existing data only. No new database tables are needed.

**Computation logic:**

Fetch the following from existing tables:
- `projectsTable`: budget, budgetedHours, trackedHours, startDate, dueDate, status
- `tasksTable` WHERE projectId = id: total count, count where status IN ('Completed','Done'), count where status IN ('On Hold','Blocked'), count where dueDate < today AND status NOT IN ('Completed','Done','Canceled')
- `allocationsTable` WHERE projectId = id AND endDate >= today: for each userId, sum hoursPerWeek; compare to a standard 40h/week to detect over-allocation (hoursPerWeek > 40 across all projects for that user — approximate by querying total allocated hoursPerWeek for each userId across all active allocations)
- `invoicesTable` WHERE projectId = id: sum of total for status IN ('Paid','Approved') = spent amount

Compute these five sub-scores (each 0–100, higher = healthier):

```
1. budgetScore:
   - If budget = 0 or null → 100 (not applicable)
   - pctConsumed = spentAmount / budget
   - taskPctComplete = completedTasks / totalTasks (or 0 if no tasks)
   - burnEfficiency = taskPctComplete / pctConsumed (ratio of work done vs money spent)
   - If pctConsumed <= 0.5 → 100
   - If pctConsumed > 1.0 → 0
   - Else: score = (1 - pctConsumed) * 100 * (burnEfficiency capped at 1.2) — clamp 0–100

2. scheduleScore:
   - If no tasks → 100
   - overdueRatio = overdueTasks / totalTasks
   - score = (1 - overdueRatio) * 100 — clamp 0–100

3. completionScore:
   - If totalTasks = 0 → 100
   - today relative to project timeline: elapsed = (today - startDate) / (dueDate - startDate) clamped 0–1
   - expectedComplete = elapsed * 100
   - actualComplete = (completedTasks / totalTasks) * 100
   - delta = actualComplete - expectedComplete
   - If delta >= 0 → 100
   - If delta < -40 → 0
   - Else: score = 100 + (delta * 2.5) — clamp 0–100

4. resourceScore:
   - overAllocatedCount = count of users on this project whose total hoursPerWeek across all active allocations > 44 (allow 10% flex)
   - totalAllocated = count of unique users on this project
   - If totalAllocated = 0 → 100
   - ratio = overAllocatedCount / totalAllocated
   - score = (1 - ratio) * 100 — clamp 0–100

5. blockerScore:
   - If totalTasks = 0 → 100
   - blockerRatio = onHoldTasks / totalTasks
   - score = (1 - blockerRatio * 3) * 100 — clamp 0–100
```

Compute weighted overall score:
```
overallScore = (
  budgetScore * 0.30 +
  scheduleScore * 0.25 +
  completionScore * 0.25 +
  resourceScore * 0.10 +
  blockerScore * 0.10
)
Round to nearest integer, clamp 0–100.
```

Determine label:
- 75–100 → "On Track"
- 50–74 → "At Risk"
- 0–49  → "Critical"

Return this JSON response:
```json
{
  "projectId": 1,
  "overallScore": 72,
  "label": "At Risk",
  "subScores": {
    "budget": 65,
    "schedule": 80,
    "completion": 70,
    "resource": 90,
    "blocker": 100
  },
  "inputs": {
    "totalTasks": 24,
    "completedTasks": 10,
    "overdueTasks": 3,
    "onHoldTasks": 1,
    "budget": 120000,
    "spentAmount": 85000,
    "overAllocatedUsers": 1,
    "totalAllocatedUsers": 3,
    "daysRemaining": 21,
    "projectDurationDays": 90
  }
}
```

Register this route in `artifacts/api-server/src/routes/index.ts` alongside the existing project routes. Do not remove or change any existing route registrations.

### SECTION B — Backend: AI Narrative Endpoint

**File to create:** `artifacts/api-server/src/routes/projectHealthNarrative.ts`

Create a new POST endpoint: `POST /api/projects/:id/health-narrative`

This endpoint accepts the health score payload in the request body (the same shape returned by Section A) and calls the Anthropic API to generate a narrative.

The Anthropic API key is already available as the environment variable `ANTHROPIC_API_KEY`.

Call: `POST https://api.anthropic.com/v1/messages`

Headers:
```
x-api-key: process.env.ANTHROPIC_API_KEY
anthropic-version: 2023-06-01
content-type: application/json
```

Body:
```json
{
  "model": "claude-sonnet-4-20250514",
  "max_tokens": 120,
  "messages": [
    {
      "role": "user",
      "content": "You are a senior project manager reviewing the health of a professional services project. Based on the following data, write exactly 2–3 sentences (maximum 60 words total) explaining the project's health status in plain business English. Be specific about the numbers. Do not use bullet points. Do not mention score numbers.\n\nProject health data:\n- Overall status: {label}\n- Budget consumed: {pctBudget}% with {pctComplete}% of tasks complete\n- Overdue tasks: {overdueTasks} of {totalTasks}\n- Blocked tasks: {onHoldTasks}\n- Over-allocated team members: {overAllocatedUsers} of {totalAllocatedUsers}\n- Days remaining: {daysRemaining}\n\nWrite the narrative now:"
    }
  ]
}
```

Replace placeholders with actual values from the request body before sending.

Timeout: if the Anthropic API does not respond within 6 seconds, return:
```json
{ "narrative": null, "error": "narrative_unavailable" }
```

On success, return:
```json
{ "narrative": "The generated narrative text here." }
```

This endpoint requires the same view permission as the health score endpoint. Register it in `artifacts/api-server/src/routes/index.ts`.

### SECTION C — Frontend: ProjectHealthBanner Component

**File to create:** `artifacts/businessnow/src/components/project-health-banner.tsx`

This component renders a horizontal banner card. It:
1. Calls `GET /api/projects/:id/health` on mount using `useQuery` with a 5-minute stale time (`staleTime: 5 * 60 * 1000`). Cache key: `['project-health', projectId]`.
2. When the health score loads, immediately calls `POST /api/projects/:id/health-narrative` with the score payload using a separate `useQuery`. Cache key: `['project-health-narrative', projectId]`.
3. Renders a banner with:
   - Left side: A circular or ring score indicator showing the numeric score (e.g., "72") in large text
   - Center: The status label ("On Track" / "At Risk" / "Critical") as a badge, plus the AI narrative below it in small muted text
   - Right side: Five sub-score pills in a row (Budget, Schedule, Completion, Resource, Blockers), each showing its label and score
   - A subtle "Refresh" icon button (top-right corner of the banner) that invalidates both cache keys on click
4. Color scheme:
   - On Track: green border + green badge (`border-green-200 bg-green-50`)
   - At Risk: amber border + amber badge (`border-amber-200 bg-amber-50`)
   - Critical: red border + red badge (`border-red-200 bg-red-50`)
5. Loading state: show a Skeleton placeholder the same height as the banner (approximately 80px tall)
6. Error state: show nothing (do not render the banner if the health API fails — fail silently)
7. Narrative loading: while the narrative is loading, show a subtle animated "..." text in its place. If narrative returns `null` (error), show nothing in that spot — the score and sub-scores still display.

Use only existing UI components: Card, CardContent, Badge, Skeleton, Progress, Button, Tooltip from the existing `@/components/ui/` directory. Do not install new dependencies.
